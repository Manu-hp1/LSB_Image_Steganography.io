A secure data concealment system built in C that embeds confidential messages inside 24-bit BMP images using precise bit-level manipulation.

Most people look at an image and see colors. This project sees binary. Every pixel holds 3 bytes — red, green, blue. Change the last bit of each and the image looks identical. But the message is there, hidden in plain sight.

The LSB Technique

Every 24-bit BMP pixel stores 3 bytes for Red, Green and Blue. Changing only the least significant bit of each byte causes less than 0.4% color change — completely invisible to the human eye.

01
Read the BMP Image
The BMP file is opened and the 54-byte header is validated for format, dimensions and bit depth. Only 24-bit BMP images are accepted.
fread(&header, sizeof(BMPHeader), 1, fp)
02
Embed Message Bits
Each character is converted to 8 bits. Each bit is embedded into the LSB of one pixel byte using AND masking to clear and OR to set.
pixel = (pixel & 0xFE) | (bit & 0x01)
03
Extract Hidden Data
During decoding, the LSB of each pixel byte is extracted and assembled back into characters. Null terminator marks end of message.
bit = pixel & 0x01

Encode & Decode

Two separate modes — encoding hides the message, decoding reveals it. Both work entirely through the command line.

ENCODE — HIDE MESSAGE
1
Open source BMP and validate 24-bit format
fopen() + header validation
2
Check image capacity against message size
capacity = (width × height × 3) / 8
3
Convert each character to 8 bits
bit = (ch >> i) & 1
4
Embed each bit into LSB of pixel byte
pixel = (pixel & 0xFE) | bit
5
Write modified pixels to output BMP
fwrite() to output.bmp
LSB LAYER
DECODE — REVEAL MESSAGE
1
Open the encoded BMP image
fopen(encoded_image, "rb")
2
Read pixel bytes one by one
fread(&pixel, 1, 1, fp)
3
Extract the LSB from each pixel byte
bit = pixel & 0x01
4
Assemble 8 bits back into one character
ch |= (bit << i)
5
Stop at null terminator and print message
if (ch == '\0') break

What It Does

🔐
Invisible Data Hiding
Modifies only the least significant bit of each pixel byte. Visual difference is less than 0.4% — completely undetectable to the human eye.
✅
BMP Header Validation
Validates the 54-byte BMP header before processing. Rejects non-BMP files, non-24-bit formats and corrupted headers automatically.
⚡
Bitwise AND & OR Masking
AND with 0xFE clears the LSB. OR with the message bit sets it. Two operations control the entire system.
📁
File I/O with fread & fwrite
Efficient binary file handling for reading pixel data and writing the modified output BMP without any data loss.
🧱
Modular C Architecture
Clear separation of encoding, decoding, validation and file handling modules. Each function has a single responsibility.
💻
Command-Line Interface
Runs entirely from the terminal. No dependencies beyond GCC. Simple arguments control encode and decode mode.

Built With

LANGUAGE
C Programming
Pure C with no external libraries. Every operation — file reading, bit manipulation, memory handling — done from scratch.
TECHNIQUE
Bit Manipulation
AND masking with 0xFE to clear LSB. OR to embed message bit. Shift operators decompose characters into individual bits.
FORMAT
24-bit BMP
Uncompressed image format. Each pixel uses 3 bytes for RGB. No compression means zero data loss during embedding.
I/O
File Handling
fopen, fread, fwrite and fclose for binary file operations. BMP header preserved exactly and pixel data modified byte by byte.
MEMORY
Pointers & Structs
C structures map directly to the BMP header layout. Pointer arithmetic navigates pixel data efficiently without unnecessary copying.
PLATFORM
Linux / GCC
Compiled with GCC using -Wall -Wextra flags for zero warnings. Developed and tested on Linux WSL environment.
